# 微信小程序迁移说明

## 当前网页版架构

```text
web/index.html       # 页面结构
web/css/style.css    # 页面样式
web/js/data.js       # 内容数据
web/js/storage.js    # 本地存储
web/js/app.js        # 交互逻辑
```

第一版是纯静态网页，不需要服务器，不需要数据库，不需要 API。

## 小程序迁移思路

### 1. 数据迁移

把 `web/js/data.js` 拆成小程序数据模块：

```text
miniprogram/data/knowledge.js
miniprogram/data/formulas.js
miniprogram/data/exercises.js
miniprogram/data/lifeCases.js
miniprogram/data/pitfalls.js
```

### 2. 页面拆分

网页版单页结构可以拆成小程序页面：

```text
pages/index/        首页和进度
pages/map/          知识图谱
pages/formulas/     公式速查
pages/practice/     分层练习和抽卡
pages/life/         生活化解读
pages/mistakes/     避坑指南
pages/notes/        错题备注
```

### 3. 存储方式

网页版使用：

```js
localStorage
```

小程序改成：

```js
wx.getStorageSync
wx.setStorageSync
```

数据结构可以保持不变。

### 4. 是否需要后台

第一版小程序仍然不需要后台。所有题库内置，进度存在本机。

什么时候需要后台？

- 多设备同步；
- 家长手机查看孩子进度；
- AI 自动生成同类题；
- 自动生成周报；
- 账号体系和云端错题本。

如果只给自己孩子使用，建议先不接后台，避免复杂度过高。

## 推荐技术路线

### 阶段一：网页版验证内容

先确认孩子愿不愿意用、哪些模块有效、题目难度是否合适。

### 阶段二：小程序离线版

用微信开发者工具导入 `miniprogram/`，把网页版内容迁移为小程序页面。

### 阶段三：AI/API增强版

再考虑接入后端服务，生成变式题和家长周报。
